MERN Roadmap

HTML and CSS

The foundation of any website has to be HTML and CSS. But why start with them? Because for any website you create, you need to understand the basics correctly. So, you must know Flexbox and CSS Grid, and how to make websites responsive on all devices, and such.

JavaScript

The language on which everything in the MERN stack is literally built. You'll need to learn the basics and add on ES6+ to gain speed in your work and understand how to handle Async/Await and Promises.

React.js

React is a frontend framework. Try to focus on components, state management, and hooks like useState and useEffect, among others.

Node.js

Node.js is what runs JavaScript on the server. You need to learn how to build APIs and communicate between the frontend and backend. The basics of Node.js will enable you to build a fast server, God willing.

Express.js

Express.js is a library that will simplify writing backend code. You'll be able to build RESTful APIs and handle user requests efficiently

MongoDB

MongoDB is a NoSQL database where you can easily store data and perform queries. You need to understand CRUD (Create, Read, Update, Delete) operations to manage data effectively as a data administrator.

Mongoose

You should be aware that MongoDB isn't always the easiest to manage directly. That's why Mongoose is a library that helps you organize and handle data in a more structured and easier way.

JWT Authentication

The most important part of the job is securing your website properly. This is where JWT (JSON Web Tokens) comes into play, providing users with a secure login method and ensuring your data is protected.

Git And GitHub

Anyone in this field must be familiar with Git. Not just for commits, but also for managing branches while working with the team.

Testing

Learn a bit about testing libraries like Jest and Mocha, as they are crucial when it comes to launching your website. Testing ensures your application is reliable, catches bugs early, and maintains high-quality code. These tools help automate the testing process, making it easier to identify and fix issues before deploying to production. Mastering them will improve your workflow and boost your confidence in the stability of your projects.

Deployment

After you finish your project, you need to deploy it. Heroku and Vercel are the best deployment tools from my experience, so make sure to focus on the details of environment variables and security.

Soft Skills و Agile

This is my personal suggestion, and I always say that more important than the code is teamwork. You need to learn how to work with Agile, how to communicate with your team, and to get familiar with tools like Jira and Trello. (I'll post an explanation about the Agile system soon)

Keep going, you're doing great!
